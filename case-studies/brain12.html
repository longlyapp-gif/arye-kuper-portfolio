Brain12

Mental clarity through play.

Role

End-to-end Product Designer

Scope

Game UX, Visual Language, Interaction Design

Platform

iOS, Android

The Problem

Cognitive games that confuse instead of challenge.

Brain training apps have a fundamental UX paradox: they need to challenge your thinking, but a confusing interface gets in the way of that challenge. When the UI is unclear, you're not exercising your brain — you're fighting the interface.

Brain12 had multiple game types — word, number, memory, spatial — all needing a coherent visual system that felt fun and focused, not cluttered and overwhelming.

The Approach

The interface should disappear. The thinking should remain.

Strategy — We defined one visual principle: the UI serves the cognitive task, not the other way around. Every design decision was evaluated by asking: does this help the user think, or does it distract?

Emotion — Play needs to feel safe. You need to know the rules before you can enjoy breaking them. We created clear onboarding moments for each game type that gave users confidence before the challenge began.

System — A unified graphic language: a shared color palette, iconography system, and motion principles that made different game types feel like part of the same family — even when the challenges were completely different.

Impact

A game experience that strengthens thinking instead of confusing it.

Users reported feeling challenged in the right way — by the games, not by the interface. Session length increased, and users returned with higher consistency. The clear visual language reduced onboarding time for new game types.

Takeaway

The interface that disappears is the interface that wins. In cognitive games, clarity isn't the absence of challenge — it's the condition that makes challenge possible.
Clarity is a design decision.
